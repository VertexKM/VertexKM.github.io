Internal example: Using the O Series Log File Utility

When you analyze issues for O Series on-premise customers and the O Series On-Demand team, you often request log files to help with troubleshooting. You can use a new in-house tool, the O Series Log File Reader Utility, to organize and filter those files.

How the utility works

The utility examines log files that you input and produces organized output files. Note that the utility processes only LOG and TXT files. If you input other file types, those files are passed through the utility without being processed. The utility also consolidates the input log files by application and produces a single output text file for each application. For example, if you input ten vertex-ws files, the utility produces one vertex-ws output file. The log messages in the output file are ordered by time stamp.

How to filter on errors

If you want to return more granular results, you can filter on the following error levels in the log files:

  • OPS

  • WARNING

  • ERROR

  • FATAL

The utility produces the following output files:

  • Files that contain the specified error levels.

  • A separate output file for fatal errors, such as the fatal JVM error OutOfMemoryError. The error file contains information about which log files contain the fatal errors.

Utility location and files

You can access the utility via the Customer Support team in Microsoft Teams. Navigate to the O Series - General Questions channel > Files tab > O Series - General Questions folder > Programs folder > Log File Reader folder.

Output files location and name formats

The output files are located in the same directory as the selected input files.

Output files are named in the format _LogReader.YYYY-MM-DD-<application>.txt. For example:

_LogReader.2020-05-07-EC Portal.txt

_LogReader.2020-05-07-vertex-ws.txt

Output files for fatal errors are named in the format _<errorname>Log.YYYY-MM-DD.txt. For example:

_OutOfMemoryLog.2020-05-07.txt.

Running the utility

Complete the following steps to run the utility:

  1. Access the utility location in MS Teams and download the files.

  2. Double-click the log-reader-1.0.3.jar file.

    The Log Reader Utility dialog box displays.

  3. Click Select... and browse for the O Series log files you want to input.

  4. Select a file or use Shift + Click to select multiple files and then click Open.

    The Select Files field displays the files you want to process.

  5. Select the check boxes for the log error levels you want to process.

    The ERROR and FATAL check boxes are required and are selected by default.

  6. Click Process Log Files.

    The Success dialog box displays when the process is done.

  7. Click OK.

  8. Navigate to the directory where the input files are located and review the new output files.
