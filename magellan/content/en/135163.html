Internal example: New Vertex® Indirect Tax Reporting & Analysis (VRA) Frequently Asked Questions

The new VRA is a reporting tool that uses data from Vertex® Indirect Tax O Series® to create standard and custom reports. Vertex is evolving this product to leverage the capabilities of the former Data Extract feature of Vertex O Series and to no longer require an IBM Cognos deployment.

How do I access the new VRA?

The VRA menu option in the O Series user interface takes customers to the new VRA. To access the new VRA, navigate to Reporting > VRA > Extract Definition.

Is the new VRA build into O Series or is it a separate installation?

The new VRA reporting tool is part of O Series. It uses the existing Data Extract feature that is familiar to customers. Customers do not need to take any additional steps to use the new VRA. They do not need to install and configure IBM Cognos or other software.

Is the Extract Definitions report enhanced or updated?

No, the report has not changed. In 9.0 SR1, the Extract Definitions report is the same report that customers currently know and use. Future releases will update the Extract Definitions report with additional functionality. Customers can continue to use the Extract Definitions, Extract Results, and Sample Extracts options on the VRA submenu for standard data extract reporting.

Why is the new tool named VRA and not something different and more distinctive?

For contractual reasons, it is necessary for Vertex to continue to use the name VRA.

What do on-premise customers need to install to use the new VRA?

On-premise customers do not need to install any extra software. The new VRA reporting tool uses the existing data extract functionality that is included in the standard Vertex O Series 9.0 installation. The only VRA changes of note in 9.0 SR1 are the renamed menu options.

What is the road map for the new VRA?

Vertex plans to build out the reporting tool to provide enhanced reporting capabilities. The first changes are delivered in the Vertex O Series 9.0 SR1 release, where the Data Extract menu option is renamed to be VRA and the VRA menu option is renamed to be Legacy VRA. Future releases of O Series will deliver additional updates.

Going forward, Vertex wants to identify functionality that is used with Legacy VRA and add equivalent functionality to the new VRA reporting tool where applicable. This effort includes adding new standard reports that can extract unique, useful data that is valuable to tax professionals. Vertex anticipates that more customers will use the new VRA tool going forward.

Is there a list of upcoming changes for the new VRA?

There is no available list of planned changes at this time. New functionality and enhancements will be identified through suggestions and input from internal and external customers. Vertex wants to collect information about how customers use Cognos reports. The intent is to identify and analyze Cognos capabilities that are not available in the current O Series reporting and data extract functionality.

Note: Contact Jacques Murphy with information about how customers use Legacy VRA. He is collecting data about how customers create, set up, and run reports, as well as the types of reports they use with Cognos that are not currently available in the new VRA data extract reporting.
