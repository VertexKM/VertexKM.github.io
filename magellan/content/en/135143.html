External example: What types of data can I import into Global Compliance?

There are two types of imports for your data: imports from O Series sources, and imports from non-Vertex sources.

When you import Vertex O Series data into Global Compliance, you use the predefined Vertex Import setup for importing VAT Returns Export data files that were generated from Vertex O Series. Because the field mappings are automatically defined, you cannot change the mappings to make data modifications to the Vertex Import setup.

For a non-Vertex data import, you must complete the following tasks:

  • Prepare your files from a non-Vertex source.

  • In the Vertex Returns desktop application:

    • Create an import setup and map import file fields to Global Compliance fields.

    • Add rules as needed to modify imported data or to map country-specific fields.

    • Create Scenario Tax Code mappings to transform tax codes to Global Compliance tax codes.
