Directory structure

The Your Vertex product directory structure controls access to files. For certain Your Vertex product files, the O Series directory structure limits file access to the partition in which they are created. Other files are stored in a subdirectory of the vertexroot directory and can be accessed from any partition in the instance. In a multitenant deployment, access to partition-specific files is limited to users that have permissions for that partition.

The following tables specify which files are available to all partitions in an O Series instance and which files are partition-specific.

Files that are available to all partitions

The directories listed in the following table are created under the directory that is specified as your vertexroot directory during the O Series installation process. In a multitenant environment, these directories are shared among tenants.

Directory

Contains

api

The vertex-api.zip file, which includes sample Java code and various scripts that you can use to build and test your deployment.

bin

Batch files that can be used to support O Series.

config

Files that are required to configure O Series. The Vertex configuration file, vertex.cfg, resides in this directory. This file contains parameters that are used to configure O Series to your business needs.

data\admin

Data subdirectories. Only users with permissions to the admin partition have access to this directory.

dataupdate\forms

PDF files for certificates.

dataupdate\reports

PDF and CSV files that are created when you run the Data Update utility.

database

Database drop scripts to remove databases.

doc

Javadoc for the Java API.

lib

Java archive (JAR) files required to run the command line utilities.

log

Log files that are generated for events that run at the system or initialization level. This does not include events that are partition-specific. Partition-specific logs are located in subdirectories of the vertexroot\log directory; these subdirectories have the same name as the partitions to which they apply (for example, vertexroot\log\partition).

manifest

Internal Extract, Transform, and Load (ETL) manifest files.

patch

An empty directory to be used for patch installation.

reportbuilder

Subdirectories for report samples, selectors, styles, and templates

schema

XML schema files.

test

Files that are used to test the web services and sample code.

web

Web Archive (WAR) files for the web services and Vertex Central servlet components.

Files that are limited to a partition

Each partition has a set of directories for the data files that are listed in the following table. When you create a partition, the following subdirectories are created under the vertexroot\data\partition directory. A configuration parameter - com.vertexinc.dataDirectory.location - in the vertex.cfg file enables you to designate a different name for the partition-specific subdirectories. Refer to the Configuration Parameters Reference Guide for more information about this parameter.
