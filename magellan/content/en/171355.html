Site permissions and permission groups

Site permissions control what users can see and do on the site. Permission levels are groupings of permissions that give a user specific access to the site. The following are the standard permission levels:

Permission

Descriptin

Full Control

The user has no restrictions on the site. This is typically reserved for the Owner permission group.

Design

The user can view, add, update, delete, approve, and customize content and site elements, such as pages.

Edit

The user can add, edit and delete lists; can view, add, update and delete list items and documents. This is typically used for the Member permission group.

Contribute

The user can view, add, update, and delete list items and documents.

Read

The user can view pages and list items and download documents. This is typically used for the Visitor permission group.

A permission group is a group that includes defined site permissions. For example, the site owners group may have the Full Control site permissions while the site visitors group may only have the Read site permissions. The following are the standard permission groups:

Permission Group

Descriptin

Owner

Site Owners have full control of the site. They can add, edit, and delete content as well as users and security groups. Limit the number of members in the Owner group to only those individuals who must have this access. Owners have the Full Control permission level.

Member

Site Members have full control of the content. They can add, edit, and delete the content. Use this group for the majority of the users. Members have the Edit permission level.

Visitor

Site Visitors have read-only access to the content. You can use this group to give access to a document library that you use to share files with other departments. Visitors have the Read permission level.

If needed, you can assign custom permissions to specific elements of a SharePoint site, such as a document library. This enables you to restrict or enable access at a more granular level. For example, you can enable access to a document library without enabling access to the entire site. This is useful if you want someone in another department to be able to view and share files without having access to the entire site. Another example is to restrict access to a library. This is useful if you have proprietary or private files that you do not want to share with everyone who has access to the site.

Note: If custom permissions are needed, it is recommended that you use them at the document library level and not the document level. It is much harder to maintain custom permissions for individual documents.

Add, edit, or delete site permissions

Complete the following to work with site permissions:

  1. (Optional) To add a site permission, do the following:

    1. On the site home page, click the Settings icon () and select Site permissions > Advanced permissions settings.

    2. In the ribbon, select Permission Levels. The Permission Levels page opens.

    3. Click Add a Permission Level. The Add a Permission Level page opens.

    4. Enter a name and description of the permission level in the Name and Description fields.

    5. Under List Permissions, select the desired permissions.

    6. Click Create.

  2. (Optional) To edit a site permission, do the following:

    1. On the site home page, click the Settings icon () and select Site permissions > Advanced permissions settings.

    2. In the ribbon, select Permission Levels. The Permission Levels page opens.

    3. Select the permission to edit. The Edit Permission Level page opens.

    4. Make the desired changes to the permission level.

    5. Click Submit.

  3. (Optional) To delete a site permission, do the following:

    1. On the site home page, click the Settings icon () and select Site permissions > Advanced permissions settings.

    2. In the ribbon, select Permission Levels. The Permission Levels page opens.

    3. Select the check box next to the permission to delete.

    4. Select Delete Selected Permission Levels.

      Note: You can delete only custom permission levels. You cannot delete the default levels.

Add, edit, or delete permission groups

Complete the following to work with permission groups:

  1. (Optional) To add a permission group, do the following:

    1. On the site home page, click the Settings icon () and select Site permissions > Advanced permissions settings.

    2. In the ribbon, click Create Group. The Create Group page opens.

    3. Configure the group settings.

    4. Click Create.

  2. (Optional) To edit or delete a permission group, do the following:

    1. On the site home page, click the Settings icon () and select Site permissions > Advanced permissions settings.

    2. Select the permission group to edit. The <permission group name> page opens.

    3. Select Settings > Group Settings. The Change Group Settings page opens.

    4. Configure the group settings.

    5. Click OK.

    6. (Optional) Click Delete to delete the permission group.

      Note: You can edit only custom permission groups. You cannot edit the default permission groups.
